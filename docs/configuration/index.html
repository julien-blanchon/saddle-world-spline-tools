Frame-mode guidance

ModeUse whenCost / tradeoff
FixedUpYou need a stable authored up axis and do not want twist transportCheapest
FrenetYou want curvature-driven normals for debuggingCan flip or become unstable on straight sections
ParallelTransportYou want low-twist placement or sweeps with simpler mathRobust and fast
RotationMinimizingYou want the best production default for roads, tubes, and loopsSlightly more work than transport, but still CPU-cheap

Performance Notes

  • The dominant cost is samples_per_segment * segment_count.
  • Tube and dense custom sweeps scale with path_sample_count * cross_section_point_count.
  • TileByWorldDistance is effectively free compared with sweep geometry.
  • RMF and transport frames are inexpensive compared with generating a dense mesh.
  • The runtime rebuilds only the affected spline entity, but each dirty mesh rebuild still regenerates the full mesh for that entity in v0.1.
